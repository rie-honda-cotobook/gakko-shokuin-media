# 大学職員転職.com 制作過程メモ

ここまでの制作の流れと、運用に必要な情報を簡潔にまとめたメモです。

---

## 1. 何を作ったか

- **公開サイト** … 大学職員向けメディア（TOP・記事・求人一覧・運営情報など）
- **管理画面** … 記事の入稿・監視URLの登録・監視イベントの確認
- **監視システム** … 登録した大学の採用ページを定期チェックし、変更があれば通知
- **通知** … Chatwork（無料プラン対応）
- **求人一覧** … 管理画面の「監視URL」（有効なものだけ）を本番の求人ページに表示

---

## 2. 技術構成

| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js（App Router）・TypeScript |
| DB | PostgreSQL（Neon）／Prisma |
| 認証 | NextAuth（Credentials・管理用） |
| ホスティング | Vercel |
| 定期実行 | GitHub Actions（cron で毎日1回） |
| 通知 | Chatwork API |

---

## 3. 制作の主な流れ

1. **ローカル開発**
   - `npm run dev` で起動。`.env` に DB・認証・Chatwork 等を設定。
   - 記事・求人・監視URLの画面とAPIを実装。

2. **DB**
   - Neon で PostgreSQL を作成し、`DATABASE_URL` を取得。
   - `npx prisma db push` でテーブル作成（Article, Job, MonitorPage, MonitorEvent 等）。

3. **本番デプロイ（Vercel）**
   - GitHub にリポジトリを push。Vercel でリポジトリを連携し、自動デプロイ。
   - Vercel の Environment Variables に本番用の値を設定（DATABASE_URL, NEXTAUTH_*, ADMIN_*, CHATWORK_*, SITE_URL, MONITOR_RUN_TOKEN 等）。

4. **監視の自動化**
   - 通知先を Slack から Chatwork に変更（`src/lib/monitor/slack.ts` で Chatwork 優先）。
   - `.github/workflows/monitor.yml` で「毎日1回（日本時間 9:00 頃）」実行するよう設定。
   - GitHub の Actions 用 Secrets に `SITE_URL` と `MONITOR_RUN_TOKEN` を登録。
   - Vercel にも `MONITOR_RUN_TOKEN` を同じ値で登録（API 認証用）。

5. **求人一覧の表示元の統一**
   - 本番の求人一覧を「Job テーブルのダミー」ではなく「管理画面の監視URL（有効な MonitorPage）」から表示するよう変更（`src/lib/repositories/job.ts`）。
   - 管理画面で登録・有効化した大学だけが、本番の求人ページに表示される。

## 5. 運用でよく使う操作

- **記事の追加・編集** … 管理画面にログイン → 記事 → 新規 or 編集。
- **監視対象の追加** … 管理画面 → 監視URL → CSVインポート（形式：大学名,URL,エリア）または「追加」で1件ずつ。
- **通知の確認** … Chatwork の指定ルームで「大学採用ページの更新を検知」を確認。必要ならリンクから「確認済み」に。
- **本番の反映** … コードを変えた場合は `git add` → `git commit` → `git push origin main`。Vercel が自動で再デプロイ。
