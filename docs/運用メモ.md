# 大学職員転職.com 運用メモ

サイトの見方・管理画面・ターミナル操作をまとめたメモです。

---

## 1. ページ（サイト）の情報

### 見るためのURL

- 開発中は **ターミナルに表示されるポート** を使う。
- 例: `Local: http://localhost:3004` と出ていれば → **http://localhost:3004**

| ページ | URL（ポート3004の例） |
|--------|------------------------|
| トップ | http://localhost:3004 |
| 求人一覧 | http://localhost:3004/jobs |
| 記事一覧 | http://localhost:3004/articles |
| 対策ガイド | http://localhost:3004/career-guide |
| 著者一覧 | http://localhost:3004/authors |
| 運営・お問い合わせ等 | http://localhost:3004/about など |

※ **3004** の部分は、そのときターミナルに表示されている番号に置き換える（3000, 3001, 3002... のどれか）。

---

## 2. 管理画面（入稿用）の情報

### ログインURL

- **http://localhost:〇〇〇〇/admin/login**
- 〇〇〇〇は、トップページを開くときと同じポート（ターミナルに表示されている番号）。

### ログインに使う情報

- **メール** … `.env` の `ADMIN_EMAIL` に書いた値
- **パスワード** … `.env` の `ADMIN_PASSWORD` に書いた値

初期の例（.env を変えていなければ）:
- メール: `admin@example.com`
- パスワード: `change-me-secure-password`

※ 本番や共有する環境では、必ず別の値に変更すること。

### 管理画面の主なページ

| 用途 | URL（ポート3004の例） |
|------|------------------------|
| ログイン | http://localhost:3004/admin/login |
| ダッシュボード | http://localhost:3004/admin |
| 記事一覧 | http://localhost:3004/admin/articles |
| 新規記事 | http://localhost:3004/admin/articles/new |
| 著者一覧 | http://localhost:3004/admin/authors |
| メディア | http://localhost:3004/admin/media |

### .env の場所と主な項目

- **場所**: プロジェクト直下の **`.env`**（`gakko-shokuin-media` フォルダの中）
- 管理画面に関係する主な項目:
  - `ADMIN_EMAIL` … ログイン用メール
  - `ADMIN_PASSWORD` … ログイン用パスワード
  - `NEXTAUTH_SECRET` … 認証用（32文字以上のランダムな文字列）
  - `NEXTAUTH_URL` … 開発時は `http://localhost:3000` のままでOK

**.env を書き換えたら、必ずサーバーを一度止めてから `npm run dev` で再起動する。**

## 3. ターミナルに入れる情報（コマンド）

### サイトを見るとき・入稿するとき（毎回やること）

1. **Cursor でターミナルを開く**  
   - メニュー「Terminal」→「新しいターミナル」

2. **プロジェクトフォルダに移動**
   ```bash
   cd /Users/tanakarie/Downloads/有価証券報告書/gakko-shokuin-media
   ```

3. **開発サーバーを起動**
   ```bash
   npm run dev
   ```

4. **表示を確認**
   - `Local: http://localhost:3004` のように **番号** が表示される
   - この番号（例: 3004）を使って、ブラウザで次のURLを開く:
     - トップ: **http://localhost:3004**
     - 管理ログイン: **http://localhost:3004/admin/login**

5. **サーバーを止めるとき**
   - ターミナルで **Ctrl + C**（Mac なら Control + C）

※ ターミナルでサーバーが動いているあいだは、**ここに別のコマンドを入力しない**。止めてから次のコマンドを打つ。

---

### よく使うコマンド一覧

| やりたいこと | コマンド |
|--------------|----------|
| サイトを起動する | `cd /Users/tanakarie/Downloads/有価証券報告書/gakko-shokuin-media` のあと `npm run dev` |
| サーバーを止める | ターミナルで **Ctrl + C** |
| DB を初期化したい（スキーマ反映） | `npx prisma db push` |
| サンプルデータを入れる | `npm run db:seed` |
| パッケージを入れ直す | `npm install` |

※ `npm run dev` の前には、必ず `cd` でプロジェクトフォルダに移動する。

## 4. よくあるトラブル

| 症状 | 対処 |
|------|------|
| 管理画面で「メールアドレスまたはパスワードが正しくありません」 | .env の ADMIN_EMAIL / ADMIN_PASSWORD を確認。サーバーを Ctrl+C で止めてから `npm run dev` で再起動してから再度ログイン。 |
| ページが開かない・アクセスできない | ターミナルの **Local: http://localhost:〇〇〇〇** の番号と、ブラウザのURLの番号が同じか確認。 |
| ポートが 3001, 3002... とずれている | 3000 が別のアプリで使われているため。ターミナルに表示されている番号を使えばOK。 |
| .env を変えたのに反映されない | サーバーを一度止めて（Ctrl+C）、もう一度 `npm run dev` で起動し直す。 |
