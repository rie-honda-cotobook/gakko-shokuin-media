# 管理画面の操作手順（入稿〜公開予約〜公開）

## 1. ログイン

1. `/admin/login` にアクセス
2. `.env` に設定した `ADMIN_EMAIL` と `ADMIN_PASSWORD` を入力
3. 「ログイン」で `/admin` ダッシュボードへ

## 2. 新規記事の作成

1. 左メニュー「記事」→「＋ 新規」、または「新規作成」から `/admin/articles/new` へ
2. **必須項目**
   - タイトル
   - スラッグ（「自動生成」でタイトルから生成可能）
   - SEO説明（120文字目安）
   - 著者（プルダウンで選択）
3. 本文は Markdown で入力。「テンプレ挿入」で導入・結論・理由・手順・FAQ・まとめのひな形を挿入可能
4. 任意：要約・カテゴリ・タグ・SEOタイトル・OGP画像URL・FAQ（JSON）・参考リンク（JSON）

## 3. 下書き保存

- 「下書き保存」をクリック
- ステータスが「下書き」のまま DB に保存され、一覧の「下書き」件数に反映
- 公開されない

## 4. 公開予約（scheduled）

1. ステータスで「公開予約」を選択
2. 「公開日時」に日時を指定（datetime-local）
3. 「公開予約」ボタンをクリック
- ステータスが「予約」で保存されます。**指定した公開日時を過ぎると、一覧・記事詳細ともに自動で「公開済み」として表示**されます（cron は不要です）。

## 5. 公開する

1. ステータスで「公開」を選択（またはそのまま）
2. 「公開する」ボタンをクリック
- 即時で `status: published`、`publishedAt` が設定され、トップ・記事一覧・sitemap に表示されます

## 6. 公開後の更新

1. `/admin/articles` で該当記事の「編集」をクリック
2. 内容を修正
3. 「下書き保存」「公開予約」「公開する」のいずれかで保存
- 「公開する」で上書き保存すると、記事詳細の「更新日」が変わります（`updatedAt` 表示）

## 7. ダッシュボードの見方

- **今月の公開記事**：当月に `publishedAt` が入った記事の本数（月10本運用の目安）
- **下書き**：status が draft の件数
- **公開予約**：status が scheduled の件数
- **直近の公開記事**：公開日時の新しい順 5 件（編集リンク付き）

## 8. 著者・メディア

- **著者**：`/admin/authors` で一覧→「編集」で名前・肩書き・プロフィール・監修フラグなどを変更
- **メディア**：現状は `/public` に画像を置き、記事の OGP や本文で URL を指定。UploadThing 等は後から差し替え可能な構成です。
