# 入稿用ページ（/admin）が開けないとき

## 1. 使うURLとポートを確認する

- ターミナルで `npm run dev` を実行したあと、**どのポートで起動したか**を確認する。
- 表示例:
  - `Local: http://localhost:3000` → **http://localhost:3000/admin/login**
  - `Port 3000 is in use, trying 3001` のとき → **http://localhost:3001/admin/login**

**必ずターミナルに表示されている方のポート（3000 または 3001）を使う。**

---

## 2. 入稿用の正しいURL

| 開きたいもの     | URL（ポート3000の場合）     |
|------------------|----------------------------|
| ログイン画面     | http://localhost:3000/admin/login |
| ログイン後ダッシュボード | http://localhost:3000/admin |

- **http** であること
- **localhost** のスペル
- **/admin/login** まで省略せず入れる

---

## 3. まずトップが開くか確認する

- ブラウザで **http://localhost:3000**（または 3001）を開く。
- 大学職員転職.comのトップが表示される → サーバーは動いている。
- その状態で **http://localhost:3000/admin/login** を開き直す。

---

## 4. エラーが出る場合

- **真っ白な画面**  
  - ブラウザの開発者ツール（F12 または右クリック→「検証」）の「Console」タブにエラーが出ていないか確認する。
- **「このサイトにアクセスできません」**  
  - サーバーが止まっていないか確認。ターミナルで `npm run dev` を再度実行する。
- **404**  
  - URLをコピー＆ペーストで入力し直す（`/admin/login` まで含める）。

---

## 5. .env の確認

ログイン画面は開くが「ログインできない」場合は、プロジェクト直下の `.env` に次があるか確認する。

- `ADMIN_EMAIL=...`（ログインに使うメール）
- `ADMIN_PASSWORD=...`（ログインに使うパスワード）
- `NEXTAUTH_SECRET=...`（32文字以上のランダムな文字列）

`.env` を変更したら、**ターミナルで一度 Ctrl+C でサーバーを止め、もう一度 `npm run dev` で起動し直す。**
