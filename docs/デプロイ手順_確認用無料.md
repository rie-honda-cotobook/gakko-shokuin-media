# 確認用・無料で Vercel にアップする手順（初めて向け）

やったことがない方向けに、画面の流れまで含めて細かく書いています。  
**所要時間の目安：30〜45分**（アカウント作成含む）

---

## 事前に用意するもの

- **メールアドレス**（GitHub・Neon・Vercel の登録に使います）
- **このプロジェクトのフォルダ**（`gakko-shokuin-media`）が手元にあること
- **ターミナル**（Mac は「ターミナル」アプリ、Windows は「コマンドプロンプト」や「PowerShell」）

---

## 全体の流れ（5ステップ）

1. **GitHub にコードを上げる**
2. **Neon でデータベース（PostgreSQL）を1つ作る**
3. **プロジェクトを PostgreSQL 用に切り替える**
4. **Vercel でサイトをデプロイする**
5. **動作確認する**

---

# ステップ1：GitHub にコードを上げる

Vercel は「GitHub のリポジトリ」からデプロイするので、まず GitHub にコードを置きます。

## 1-1. GitHub アカウントを作る（すでにあれば飛ばしてOK）

1. ブラウザで **https://github.com** を開く
2. 右上の **Sign up** をクリック
3. メール・パスワード・ユーザー名を入力してアカウント作成
4. メール認証が求められたら、送られてきたリンクを開いて認証する

## 1-2. 新しいリポジトリを作る

1. GitHub にログインした状態で、右上の **+** → **New repository** をクリック
2. **Repository name** に `gakko-shokuin-media` など好きな名前を入力（例のままでもOK）
3. **Public** を選ぶ
4. **Add a README file** は**チェックしない**（中身は手元のプロジェクトで用意するため）
5. **Create repository** をクリック
6. 作成後、画面に **「…or push an existing repository from the command line」** と出る。その中の **2行のコマンド** をあとで使うので、この画面を開いたままにするかメモする  
   - 例：  
     `git remote add origin https://github.com/あなたのユーザー名/gakko-shokuin-media.git`  
     `git branch -M main`  
     `git push -u origin main`

## 1-3. 手元のプロジェクトを Git で管理し、GitHub にプッシュする

**初めて Git を使う場合**：ターミナルで名前とメールを1回だけ設定します。

```bash
cd /Users/tanakarie/Downloads/有価証券報告書/gakko-shokuin-media
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメール@example.com"
```

**リポジトリの初期化とプッシュ**：

1. ターミナルでプロジェクトのフォルダに移動する  
   ```bash
   cd /Users/tanakarie/Downloads/有価証券報告書/gakko-shokuin-media
   ```

2. Git の初期化（まだやっていなければ）  
   ```bash
   git init
   ```

3. すべてのファイルを一度追加する  
   ```bash
   git add .
   ```

4. コミットする  
   ```bash
   git commit -m "初回：大学職員転職.com"
   ```

5. ブランチ名を `main` にする（GitHub のデフォルトに合わせる）  
   ```bash
   git branch -M main
   ```

6. GitHub のリポジトリを「リモート」として追加する  
   - 次の `あなたのユーザー名` と `リポジトリ名` を、1-2 で作ったリポジトリに合わせて書き換える  
   ```bash
   git remote add origin https://github.com/あなたのユーザー名/gakko-shokuin-media.git
   ```

7. GitHub にプッシュする  
   ```bash
   git push -u origin main
   ```
   - 初回は GitHub のユーザー名とパスワード（またはトークン）を聞かれる。パスワードは「Personal Access Token」が必要な場合がある（GitHub の Settings → Developer settings → Personal access tokens で発行）

**注意**：`.env` にはパスワードなどが書いてあるので、Git に含めないようにします。次のコマンドで除外します（プロジェクト直下で実行）。

```bash
echo ".env" >> .gitignore
echo "node_modules" >> .gitignore
echo "*.db" >> .gitignore
git add .gitignore
git commit -m "gitignore 追加"
git push
```

もしすでに `.env` をコミットしてしまった場合は、  
`.gitignore` に `.env` を追加したうえで `git rm --cached .env` を実行し、コミット・プッシュし直してください。

# ステップ2：Neon でデータベース（PostgreSQL）を1つ作る

Vercel 上では SQLite が使えないため、クラウドの PostgreSQL（Neon）を1つ作ります。

## 2-1. Neon にサインアップする

1. ブラウザで **https://neon.tech** を開く
2. **Sign Up** をクリック
3. **Continue with GitHub** を選ぶと、GitHub 連携でアカウント作成ができる（おすすめ）
4. 許可を求められたら **Authorize** をクリック

## 2-2. プロジェクト（DB）を作る

1. ログイン後の画面で **Create a project** をクリック
2. **Project name** に `gakko-shokuin-media` など好きな名前を入力
3. **Region** で **Asia Pacific (Singapore)** または **Northeast Asia** があればそれを選ぶ（日本に近いほうが速い）
4. **Create project** をクリック
5. 作成が終わると **Connection string** が表示される  
   - 形式は次のようなものです：  
     `postgresql://ユーザー名:パスワード@ep-xxxxx.region.aws.neon.tech/neondb?sslmode=require`
6. **Copy** でこの文字列をコピーし、メモ帳などに貼って保存する（あとで `DATABASE_URL` に使います）

# ステップ3：プロジェクトを PostgreSQL 用に切り替える

手元のプロジェクトで、SQLite ではなく Neon の PostgreSQL を使うようにします。

## 3-1. Prisma の設定を PostgreSQL に変更する

1. プロジェクト内の **`prisma/schema.prisma`** を開く
2. 次の部分を探す：  
   ```prisma
   datasource db {
     provider = "sqlite"
     url      = env("DATABASE_URL")
   }
   ```
3. `provider = "sqlite"` を `provider = "postgresql"` に変更する：  
   ```prisma
   datasource db {
     provider = "postgresql"
     url      = env("DATABASE_URL")
   }
   ```
4. 保存する

## 3-2. 環境変数 .env を用意する

1. プロジェクトの **ルート**（`gakko-shokuin-media` の直下）に **`.env`** があるか確認する。なければ新規作成する
2. `.env` を開き、次のように書く（`DATABASE_URL` には 2-2 でコピーした Neon の Connection string を貼る）
   ```env
   DATABASE_URL="postgresql://xxxx:xxxx@ep-xxxx.region.aws.neon.tech/neondb?sslmode=require"
   NEXT_PUBLIC_SITE_URL=http://localhost:3000
   ```
   - 本番用の URL はあとで Vercel 側で上書きするので、ここでは `http://localhost:3000` のままでOK
   - すでに NEXTAUTH や ADMIN の行がある場合は、そのまま残して `DATABASE_URL` と `NEXT_PUBLIC_SITE_URL` だけ追加または上書きする

## 3-3. Neon にテーブルを作り、シードを流す

ターミナルでプロジェクトのフォルダにいる状態で、次を順に実行する。

1. Prisma で Neon にテーブルを作成する  
   ```bash
   npx prisma db push
   ```
   - 成功すると「Database is now in sync with your schema」のようなメッセージが出る

2. 初期データ（監修者・記事・求人など）を入れる  
   ```bash
   npx prisma db seed
   ```
   - エラーが出なければ OK

3. ローカルで動作確認（任意）  
   ```bash
   npm run dev
   ```
   - ブラウザで http://localhost:3000 を開き、表示や管理画面（/admin）が問題ないか確認する。終わったらターミナルで Ctrl+C で止める

## 3-4. 変更を GitHub にプッシュする

- **注意**：`.env` はプッシュしない（.gitignore に含めていること）。  
  `schema.prisma` の変更だけをプッシュする。

```bash
git add prisma/schema.prisma
git commit -m "本番用に PostgreSQL に切り替え"
git push
```

# ステップ4：Vercel でサイトをデプロイする

## 4-1. Vercel にサインアップする

1. ブラウザで **https://vercel.com** を開く
2. **Sign Up** をクリック
3. **Continue with GitHub** を選ぶ
4. GitHub の認証を行う

## 4-2. プロジェクトを GitHub から読み込む

1. ログイン後の画面で **Add New…** → **Project** をクリック
2. **Import Git Repository** の一覧に、GitHub の `gakko-shokuin-media` が出るので、**Import** をクリック
3. **Configure Project** の画面になる
   - **Project Name** はそのままでOK（変更してもよい）
   - **Framework Preset** は **Next.js** のまま
   - **Root Directory** は空のまま（プロジェクトがリポジトリのルートにあるため）
   - **Build and Output Settings** はそのままでOK

## 4-3. 環境変数を設定する（ここが重要）

1. **Environment Variables** のところを開く
2. 次の2つを追加する

   | Name | Value |
   |------|--------|
   | `DATABASE_URL` | ステップ2-2でコピーした Neon の Connection string（そのまま貼る） |
   | `NEXT_PUBLIC_SITE_URL` | いったん `https://プロジェクト名.vercel.app` でOK（デプロイ後に表示される URL に合わせてあとで直せる） |

3. 管理画面（/admin）を使う場合は、ローカルの `.env` にある次の値も入れておく（本番用に別の値に変えておくと安全です）
   - `NEXTAUTH_URL` → `https://あなたのプロジェクト名.vercel.app`
   - `NEXTAUTH_SECRET` → 32文字以上のランダムな文字列
   - `ADMIN_EMAIL` → 管理者のメール
   - `ADMIN_PASSWORD` → 管理者のパスワード

4. **Deploy** をクリックする

## 4-4. デプロイが終わるまで待つ

1. 画面上でビルドの進行が表示される（1〜3分程度かかることが多い）
2. **Congratulations!** と出て、**Visit** のリンクが表示されたら完了
3. **Visit** をクリックすると、公開されたサイトが開く  
   - URL は `https://gakko-shokuin-media.vercel.app` のような形（プロジェクト名によって変わる）

# ステップ5：動作確認する

1. **トップページ**が正しく表示されるか
2. **記事一覧**（/articles）が表示されるか
3. **求人一覧**（/jobs）が表示されるか
4. **監修者一覧**（/authors）が表示されるか
5. **管理画面**（/admin）にログインできるか（環境変数で設定したメール・パスワードで）

問題があれば、Vercel のダッシュボードの **Deployments** → 該当デプロイ → **Building** や **Functions** のログでエラー内容を確認できます。

# よくあるトラブル

## ビルドが失敗する

- **「DATABASE_URL is not defined」**  
  → Vercel の **Project Settings → Environment Variables** で `DATABASE_URL` が入っているか確認する。入れたあと、**Redeploy** する。
- **Prisma のエラー**  
  → ターミナルで `npx prisma generate` を実行し、`prisma/schema.prisma` の書き間違いがないか確認する。

## ページを開くとエラーになる

- **「Cannot connect to database」**  
  → Neon の Connection string が正しいか、Neon のダッシュボードでプロジェクトが有効か確認する。
- **管理画面にログインできない**  
  → Vercel の環境変数で `NEXTAUTH_URL`（本番の URL）と `ADMIN_EMAIL` / `ADMIN_PASSWORD` が正しく設定されているか確認する。

## あとからコードを直したとき

- 手元で修正したら、次のコマンドで GitHub にプッシュする  
  ```bash
  git add .
  git commit -m "〇〇を修正"
  git push
  ```
- Vercel は GitHub と連携していれば、プッシュするだけで自動で再デプロイされる

# まとめ

1. **GitHub** にリポジトリを作り、手元のコードをプッシュ  
2. **Neon** で PostgreSQL を1つ作り、Connection string をコピー  
3. **prisma/schema.prisma** を `postgresql` に変更し、`.env` に `DATABASE_URL` を設定 → `npx prisma db push` と `npx prisma db seed` を実行  
4. **Vercel** で「New Project」→ GitHub のリポジトリを選択 → 環境変数（`DATABASE_URL` と `NEXT_PUBLIC_SITE_URL` など）を設定 → Deploy  
5. 表示と管理画面で動作確認  

ここまでできれば、確認用の無料公開は完了です。
