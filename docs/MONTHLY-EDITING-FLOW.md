# 月10本の編集フロー提案（簡易）

## 前提

- 月に約10本の記事を公開する想定
- ダッシュボードの「今月の公開記事」で本数を確認

## 週単位の目安

| 週 | 目安 |
|----|------|
| 1週目 | 2〜3本公開、2本下書き作成開始 |
| 2週目 | 2〜3本公開、2本下書き作成・加筆 |
| 3週目 | 2〜3本公開、2本下書き仕上げ |
| 4週目 | 2本公開、翌月分のテーマ・下書きを少し先行 |

## 1本あたりの流れ（簡易）

1. **テーマ決定**（カテゴリ・キーワードを決める）
2. **下書き作成**  
   - 新規記事でタイトル・スラッグ・著者・SEO説明を入力  
   - 「テンプレ挿入」で骨子を入れ、本文を書く  
   - 「下書き保存」で保存
3. **推敲・FAQ・参考リンク**  
   - 編集画面で加筆・FAQ（JSON）・参考リンク（JSON）を追加  
   - 再度「下書き保存」
4. **公開**  
   - ステータス「公開」で「公開する」をクリック  
   - 必要なら「公開予約」で日時指定（cron 実装後は自動公開に連携可能）

## 運用のコツ

- 月末に「今月の公開記事」が 10 前後になるよう調整
- 下書きを常に 2〜3本持っておくと、急な空きにも対応しやすい
- 対策ガイド（`/career-guide`）のカテゴリ別一覧を意識し、偏りなくカテゴリを選ぶと内部リンクが整理される
